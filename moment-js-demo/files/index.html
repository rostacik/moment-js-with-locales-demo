demo page to show problem with localized names of days and months (since I am from Slovakia, I used our Slovak locale for demo purposes)


localoriginalfile.html - original file loaded locally with simple script tag - not working

remotefile.html - file loaded from GitHub servers - working since server explicitly states in response headers : Content-Type:application/javascript; charset=utf-8

localconvertedfile.html - file converted to UTF8 with BOM - in VS2013 you will see this in File > Advanced Save Options.... UTF8 with signature

localforcedutf.html - not converted file but with force to use specified charset in script tag
